# 网易 126 邮箱 24 小时摘要

## 适用场景
- 定时任务脚本已经输出过去 24 小时邮件的 JSON
- 需要把 JSON 转成适合 Telegram/聊天窗口阅读的中文邮件摘要
- 用户偏好“稍详细”的版本，而不是极简提醒

## 输入假设
脚本会输出类似：
- mailbox
- window
- generated_at
- count
- emails[]

每封邮件通常包含：
- date_local
- from
- subject
- preview
- body_length

## 输出格式
按下面结构输出：

1. 第一行：
   - `过去24小时邮件摘要`
   - 或直接写明时间范围（如果输入中可推断）
2. 第二行：
   - `邮件总数：N 封`
3. 按重要性分组：
   - `一、建议关注`
   - `二、通知 / 促销类`
   - `三、其他`
4. 每封邮件使用详细版格式：
   - 时间
   - 发件人
   - 主题
   - 详细摘要（2~5 句，基于 preview 提炼）
   - 结论标签：`建议关注` / `可忽略`
5. 结尾加：
   - `如果需要，我可以继续帮你读某一封的完整内容。`

## 分类规则
### 建议关注
满足任一项可归入“建议关注”：
- 安全提醒 / 新设备登录 / 验证码 / 账号风险
- 银行 / 信用卡 / 支付 / 账单 / 扣费 / 发票
- GitHub / 云服务 / 开发平台的计费规则变化
- 人工来信 / 工作事务 / 系统告警

### 通知 / 促销类
满足任一项可归入“通知 / 促销类”：
- 营销、折扣、升级优惠、广告
- 社交平台推荐好友、订阅提醒
- 与金钱、账号安全、工作任务无直接关系的常规通知

### 其他
无法明确判断且信息量较低时归为“其他”。

## 摘要写法要求
- 不要原样复述 preview 垃圾文本、HTML/CSS、模板噪音
- 要把 preview 中的关键事实提炼成人类可读语言
- 对营销类邮件，明确指出它是促销/通知，并标注可忽略
- 对安全/账单/计费变更类邮件，明确说明为什么值得关注
- 不要编造 preview 里没有出现的事实

## 无邮件时
如果 count 为 0，直接输出：
`过去24小时没有新邮件。`

## 风格
- 中文
- 清晰
- 比简版更详细，但避免冗长
- 适合 Telegram 直接阅读
